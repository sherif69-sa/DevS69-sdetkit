# Generated and sample artifacts

This directory is a committed evidence gallery. It contains generated/sample artifacts, product proof packs, adoption examples, and historical completion material.

## How to read this directory

- **Generated/sample material:** files here are examples unless a command explicitly writes to `docs/artifacts/`.
- **Runtime evidence:** current run artifacts normally live under `build/`, `.sdetkit/out/`, or `.sdetkit/maintenance/`.
- **Product proof:** current live-adoption proof is summarized in [`../live-adoption-product-proof.md`](../live-adoption-product-proof.md).
- **Artifact map:** use [`../artifact-reference.md`](../artifact-reference.md) for runtime paths, maintenance-autopilot uploads, and artifact-to-action guidance.
- **Safety posture:** samples, plans, and safe-fix evidence do not authorize mutation. Mutation requires explicit guarded policy and PR-only controls.

Do not delete historical packs during routine docs cleanup. If a pack is moved or deprecated, update the artifact reference and any links in the same change.
