# First-proof benchmark narrative

## Outcome framing

Benchmark success is measured by:

- Reduced time-to-first-proof for contributors.
- Increased SHIP-rate stability over rolling 7-run windows.
- Faster remediation on sustained NO-SHIP streaks via owner routing.

## Evidence sources

- `build/first-proof/first-proof-summary.json`
- `build/first-proof/weekly-trend.json`
- `build/first-proof/weekly-threshold-check.json`
- `build/owner-escalation-payload.json`

## Narrative template

1. Baseline period: report initial SHIP/NO-SHIP mix and top failed steps.
2. Intervention period: list lane/policy/process changes.
3. Result period: report SHIP-rate, consecutive NO-SHIP trend, and escalation response time.
